# 健壮多文档知识库系统设计方案

## 一、当前系统问题分析

### 1.1 核心问题
```
问题根因：后台处理和前台RAG使用不同的索引文件！

后台处理:
  相关文档/
    registry.json          ← 文档注册表
    retrieval_index.json   ← 向量索引
    doc_doc_xxx/
      chunks/chunks.json   ← 分块数据

前台RAG:
  backend/
    embedding_cache.json   ← 缓存
  retrieval_index.json     ← 另一个索引
```

### 1.2 具体症状
1. **索引不同步** - 后台处理的文档不会自动出现在前台RAG
2. **硬编码路径** - ragService.js 硬编码指向错误目录
3. **缓存无关联** - embedding_cache.json 不记录 documentId
4. **手动修复** - 每次都要本小姐手动修复关联

---

## 二、健壮系统设计原则

### 2.1 单一数据源 (Single Source of Truth)
```
所有组件必须读写同一个索引文件！
```

### 2.2 事件驱动同步
```
文档处理完成 → 触发事件 → 自动同步到RAG索引
```

### 2.3 统一配置管理
```
项目根目录/
  config.json              ← 统一配置文件
  .env                     ← 环境变量（API Key等）
  文档库/
    registry.json           ← 唯一注册表
    indexes/
      retrieval_index.json  ← 唯一向量索引
      embedding_cache.json ← 唯一embedding缓存
```

### 3.2 核心组件

#### 3.2.1 统一配置管理 (ProjectConfig)
```javascript
class ProjectConfig {
    // 读取配置文件
    static getConfig() {
        return JSON.parse(readFile('config.json'));
    }

    // 获取索引路径（所有服务都用这个）
    static getIndexPath() {
        return path.join(this.getConfig().paths.indexDir, 'retrieval_index.json');
    }

    // 获取缓存路径
    static getCachePath() {
        return path.join(this.getConfig().paths.indexDir, 'embedding_cache.json');
    }

    // 获取注册表路径
    static getRegistryPath() {
        return this.getConfig().paths.registryPath;
    }
}
```

#### 3.2.2 事件总线 (EventBus)
```javascript
// 文档处理完成后发布事件
eventBus.emit('document:indexed', { documentId, chunks });

// RAG服务订阅事件
eventBus.on('document:indexed', async ({ documentId, chunks }) => {
    // 重新加载索引
    await retrievalEngine.loadIndex(ProjectConfig.getIndexPath());
});
```

#### 3.2.3 索引管理器 (IndexManager)
```javascript
class IndexManager {
    // 文档添加到索引时，同步触发缓存更新
    async addDocument(documentId, chunks, documentMeta) {
        // 1. 添加到索引
        await this.addToIndex(documentId, chunks, documentMeta);

        // 2. 生成新chunks的embedding并加入缓存
        for (const chunk of chunks) {
            if (!cache.has(chunk.chunkId)) {
                const embedding = await this.generateEmbedding(chunk);
                cache.set(chunk.chunkId, embedding);
            }
        }

        // 3. 发布事件
        eventBus.emit('index:updated', { documentId });
    }

    // 文档删除时，同步触发缓存更新
    async removeDocument(documentId) {
        // 1. 从索引移除
        await this.removeFromIndex(documentId);

        // 2. 从缓存移除
        const chunkIds = this.getChunkIdsByDocument(documentId);
        for (const chunkId of chunkIds) {
            cache.delete(chunkId);
        }

        // 3. 发布事件
        eventBus.emit('index:updated', { documentId });
    }
}
```

```json
      "chunkId": 1,
      "documentId": "doc_xxx",
      "documentName": "文档名称",

      "metadata": {
        "page": 1,
        "source": "文件路径"
      }
    }
  ]

```

```json
{
  "version": "2.0",

      "documentId": "doc_xxx",
      "embedding": [0.123, ...],
      "createdAt": "2026-04-11T00:00:00Z"
    }
  }
}
```
