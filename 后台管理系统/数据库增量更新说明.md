# 数据库增量更新机制说明

## 概述

本后台管理系统实现了完整的数据库增量更新机制，支持通过前端界面直接修改数据，所有修改实时同步到主系统的数据库中。

## 核心特性

### 1. 实时同步
- ✅ 所有CRUD操作立即写入数据库
- ✅ 主系统可立即看到更新后的数据
- ✅ 无需重启服务或手动导入

### 2. 数据一致性
- ✅ 使用事务确保数据完整性
- ✅ 支持错误回滚
- ✅ 防止数据冲突和丢失

### 3. 增量更新
- ✅ 只修改变化的数据
- ✅ 不影响其他数据
- ✅ 高效且安全

## 技术实现

### 数据库连接

使用 `better-sqlite3` 直接连接SQLite数据库：

```typescript
import Database from 'better-sqlite3'

export function getDatabase(): Database.Database {
  if (!db) {
    db = new Database(dbPath)
    db.pragma('journal_mode = WAL') // 启用WAL模式提高并发性能
  }
  return db
}
```

### 事务处理

所有数据修改操作都通过事务执行：

```typescript
export function executeTransaction<T>(
  fn: (db: Database.Database) => T
): T {
  const database = getDatabase()
  const transaction = database.transaction(fn)
  return transaction(database)
}
```

### CRUD操作示例

#### 创建数据

```typescript
export async function createStudent(req: Request, res: Response): Promise<void> {
  const data = req.body
  const db = getDatabase()

  const result = db.prepare(`
    INSERT INTO students (name, student_id, class_name, major, grade, phone, email)
    VALUES (?, ?, ?, ?, ?, ?, ?)
  `).run(
    data.name,
    data.student_id,
    data.class_name,
    data.major,
    data.grade,
    data.phone,
    data.email || null
  )

  const student = db.prepare('SELECT * FROM students WHERE id = ?')
    .get(result.lastInsertRowid)

  res.json({
    success: true,
    data: student,
    message: 'Student created successfully',
  })
}
```

#### 更新数据

```typescript
export async function updateStudent(req: Request, res: Response): Promise<void> {
  const { id } = req.params
  const data = req.body
  const db = getDatabase()

  db.prepare(`
    UPDATE students
    SET name = ?, student_id = ?, class_name = ?, major = ?, grade = ?, phone = ?, email = ?
    WHERE id = ?
  `).run(
    data.name,
    data.student_id,
    data.class_name,
    data.major,
    data.grade,
    data.phone,
    data.email || null,
    id
  )

  const student = db.prepare('SELECT * FROM students WHERE id = ?').get(id)

  res.json({
    success: true,
    data: student,
    message: 'Student updated successfully',
  })
}
```

#### 删除数据

```typescript
export async function deleteStudent(req: Request, res: Response): Promise<void> {
  const { id } = req.params
  const db = getDatabase()

  db.prepare('DELETE FROM students WHERE id = ?').run(id)

  res.json({
    success: true,
    message: 'Student deleted successfully',
  })
}
```

## 安全考虑

### 1. SQL注入防护

使用参数化查询防止SQL注入：

```typescript
// ✅ 安全
db.prepare('SELECT * FROM students WHERE id = ?').get(id)

// ❌ 不安全
db.prepare(`SELECT * FROM students WHERE id = ${id}`).get()
```

### 2. 数据验证

在控制器层进行数据验证：

```typescript
if (!data.name || !data.student_id) {
  return res.status(400).json({
    success: false,
    error: 'Missing required fields',
  })
}
```

### 3. 错误处理

捕获并记录所有数据库错误：

```typescript
try {
  // 数据库操作
} catch (error) {
  console.error('Database error:', error)
  return res.status(500).json({
    success: false,
    error: 'Database operation failed',
  })
}
```

## 性能优化

### 1. WAL模式

启用WAL模式提高并发性能：

```typescript
db.pragma('journal_mode = WAL')
```

### 2. 连接池

重用数据库连接，避免频繁打开关闭：

```typescript
let db: Database.Database | null = null

export function getDatabase(): Database.Database {
  if (!db) {
    db = new Database(dbPath)
  }
  return db
}
```

### 3. 分页查询

使用分页避免一次性加载大量数据：

```typescript
export function paginate<T>(
  query: string,
  params: any[],
  page: number,
  pageSize: number
): { data: T[]; total: number } {
  const database = getDatabase()

  // 获取总数
  const countQuery = query.replace(/SELECT[\s\S]+?FROM/, 'SELECT COUNT(*) AS count FROM')
  const countResult = database.prepare(countQuery).get(...params) as { count: number }
  const total = countResult.count

  // 获取分页数据
  const offset = (page - 1) * pageSize
  const paginatedQuery = `${query} LIMIT ${pageSize} OFFSET ${offset}`
  const data = database.prepare(paginatedQuery).all(...params) as T[]

  return { data, total }
}
```
