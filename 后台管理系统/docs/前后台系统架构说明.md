## 系统架构概述

本系统由两个独立的应用组成：

```
E:\外包\教育系统智能体\          ← 前台系统（学生/老师使用）
├── frontend\                    ← 前台前端（端口3000/5174）
├── backend\                     ← 前台后端（端口3000）
│   └── services\ragService.js  ← RAG问答服务
└── retrieval_index.json        ← 索引文件（由后台系统生成）

E:\外包\教育系统智能体\后台管理系统\  ← 后台系统（管理员使用）
├── frontend\                    ← 后台前端（端口5173）
├── backend\                     ← 后台后端（端口3004）
│   └── src\services\documentProcessingService.js  ← 文档处理服务
└── docs\                        ← 文档目录
```

## 数据流向

```
┌─────────────────────────────────────────────────────────────┐
│                      文档处理流程                            │
└─────────────────────────────────────────────────────────────┘

1. 管理员上传文档
   ↓
2. 后台系统提取文档内容（Python脚本）
   ↓
3. 智能分块（根据文档类型动态调整策略）
   ↓
4. 批量向量化（调用智谱AI API）
   ↓
5. 构建索引（保存到前台系统根目录）
   ↓
6. 前台系统自动读取新索引
   ↓
7. 学生/老师提问时使用最新索引

┌─────────────────────────────────────────────────────────────┐
│                      问答流程                                │
└─────────────────────────────────────────────────────────────┘

1. 学生/老师在前台系统提问
   ↓
2. 前台系统读取索引文件（由后台系统构建）
   ↓
3. 向量化问题
   ↓
4. 检索相关文档块
   ↓
5. Rerank重排序（可选）
   ↓
6. 生成答案（调用智谱AI API）
   ↓
7. 返回答案和参考来源
```

## 索引文件路径

### 修改前的问题

**后台系统保存位置**：
```
E:\外包\教育系统智能体\后台管理系统\backend\data\knowledge_index\retrieval_index.json
```

**前台系统读取位置**：
```
E:\外包\教育系统智能体\retrieval_index.json
```

**问题**：两个路径不一致，前台系统无法读取后台系统构建的索引！ ❌

### 修改后的解决方案

**后台系统保存位置**：
```javascript
this.basePath = path.resolve(__dirname, '../../../../');
// 解析后：E:\外包\教育系统智能体\
```

**前台系统读取位置**：
```javascript
indexPath: path.join(__dirname, '../../retrieval_index.json')
// 解析后：E:\外包\教育系统智能体\retrieval_index.json
```

**结果**：两个路径指向同一个文件，前台系统可以直接使用后台系统构建的索引！ ✅

## API接口说明

### 后台管理系统API（端口3004）

#### 1. API密钥管理

```javascript
// 获取支持的模型列表
GET http://localhost:3004/api/knowledge/models

// 验证API密钥
POST http://localhost:3004/api/knowledge/validate-key
Body: { provider: 'ZHIPU', apiKey: 'xxx' }

// 获取项目配置状态
GET http://localhost:3004/api/knowledge/project-config
```

#### 2. 索引管理

```javascript
// 构建索引
POST http://localhost:3004/api/knowledge/build
Body: { documentId: 'xxx', mode: 'new' | 'add' | 'replace' }

// 获取索引状态
GET http://localhost:3004/api/knowledge/index-status

// 获取构建进度
GET http://localhost:3004/api/knowledge/progress/:taskId

// 删除文档索引
DELETE http://localhost:3004/api/knowledge/document/:documentId

// 删除全部索引
DELETE http://localhost:3004/api/knowledge/index/all
```

### 前台系统API（端口3000）

#### 1. RAG问答

```javascript
// 初始化RAG服务
POST http://localhost:3000/api/rag/initialize

// 问答接口
POST http://localhost:3000/api/rag/answer
Body: { question: '重修的条件是什么？', options: { topK: 15, minScore: 0.3 } }

// 批量问答
POST http://localhost:3000/api/rag/batch-answer
Body: { questions: [...], options: {...} }

// 获取统计信息
GET http://localhost:3000/api/rag/stats
```

## 使用流程

### 场景1：初次构建知识库

1. **启动后台管理系统**
   ```bash
   cd E:\外包\教育系统智能体\后台管理系统\backend
   npm run dev
   # 后端运行在 http://localhost:3004
   ```

2. **启动后台前端**
   ```bash
   cd E:\外包\教育系统智能体\后台管理系统\frontend
   npm run dev
   # 前端运行在 http://localhost:5173
   ```

3. **配置API密钥**
   - 访问 http://localhost:5173/knowledge-management
   - 系统自动读取前台项目的API密钥
   - 或手动输入新的API密钥

4. **上传文档**
   - 将文档放到：`E:\外包\教育系统智能体\相关文档\`
   - 在后台界面选择文档
   - 点击"构建新索引"

5. **查看构建进度**
   ```
   ████████████████░░░░░░░░ 70%

   当前步骤: 向量化中... (150/252)
   ```

6. **构建完成**
   - 索引保存到：`E:\外包\教育系统智能体\retrieval_index.json`
   - 前台系统自动可以使用

### 场景2：更新知识库

**方式1：增量添加**
- 添加新文档，不影响现有索引
- 适合：新增少量文档

**方式2：替换索引**
- 完全重建索引，自动备份旧索引
- 适合：文档内容有重大变更

### 场景3：学生/老师使用问答

1. **启动前台系统**
   ```bash
   cd E:\外包\教育系统智能体\backend
   node start.js
   # 后端运行在 http://localhost:3000
   ```

2. **访问问答界面**
   - 访问 http://localhost:3000
   - 或 http://localhost:5174

3. **提问**
   ```
   学生：重修的条件是什么？
   系统：[返回基于文档的准确答案]
   ```

## 关键特性

### ✨ 智能分块策略

系统根据文档内容自动选择分块策略：

| 文档类型 | 关键词 | 块大小 | 重叠 | 分割方式 |
|---------|--------|--------|------|----------|
| 政策文档 | 政策、规定、办法 | 800 | 150 | 按条款 |
| 手册文档 | 手册、指南、须知 | 1000 | 200 | 按章节 |
| 技术文档 | 技术、规范、操作 | 1200 | 200 | 按步骤 |

### ✨ 口语化支持

前台系统支持口语化问题处理：

```
学生输入："我挂科了咋办？"
系统转换："学生不及格怎么办？"
然后进行检索和回答
```

### ✨ Rerank优化

可选的Rerank服务，优化检索结果排序：
- 提高准确率
- 提升相关性
- 更好的用户体验
