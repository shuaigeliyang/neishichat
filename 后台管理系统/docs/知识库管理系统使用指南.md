## 文档处理流程

### 1. 准备文档

支持的文档格式：
- ✅ DOCX（推荐）
- ✅ PDF（需要额外处理）
- ✅ TXT
- ✅ Markdown

文档位置：
```
E:\外包\教育系统智能体\相关文档\
├── 2025年本科学生手册-定.docx
└── 2025年本科学生手册-定 (1).pdf
```

### 2. 智能分块策略

系统会根据文档内容自动选择分块策略：

#### 政策文档
- **特征**：包含"政策"、"规定"、"办法"等关键词
- **块大小**：800字符
- **重叠**：150字符
- **分割方式**：按条款分割

#### 手册文档
- **特征**：包含"手册"、"指南"、"须知"等关键词
- **块大小**：1000字符
- **重叠**：200字符
- **分割方式**：按章节分割

#### 技术文档
- **特征**：包含"技术"、"规范"、"操作"等关键词
- **块大小**：1200字符
- **重叠**：200字符
- **分割方式**：按步骤分割

### 3. 处理流程

```
文档上传
    ↓
提取内容（Python脚本）
    ↓
智能分析（检测类型）
    ↓
动态分块（根据内容类型）
    ↓
向量化（批量处理）
    ↓
构建索引（保存到文件）
```

## 索引管理

### 构建新索引

1. 进入"知识库配置"页面
2. 点击"构建新索引"
3. 选择文档并确认
4. 查看实时进度

**进度显示**：
```
███████████████░░░░░░░░ 70%

当前步骤: 向量化中... (150/252)
```

### 增量添加

**使用场景**：添加新文档到现有索引

1. 上传新文档
2. 点击"增量添加"
3. 系统会：
   - 提取新文档
   - 分块并向量化
   - 合并到现有索引
   - 保持旧文档不变

### 替换索引

**使用场景**：完全重建索引

1. 选择要替换的文档
2. 点击"替换索引"
3. 系统会：
   - 备份旧索引
   - 重建新索引
   - 替换旧索引

**注意**：替换操作会备份旧索引！

### 删除索引

#### 删除单个文档索引
1. 在文档列表中找到要删除的文档
2. 点击"删除"
3. 系统会：
   - 从索引中移除该文档的所有文档块
   - 更新索引文件

#### 删除全部索引
1. 进入"索引管理"页面
2. 点击"删除全部索引"
3. **警告**：此操作不可恢复！

## 故障排除

### 问题1：端口冲突

**症状**：
```
Error: listen EADDRINUSE: address already in use :::3004
```

**解决**：
```bash
# 查找占用端口的进程
netstat -ano | findstr :3004

# 终止进程（替换<PID>为实际PID）
taskkill //F //PID 3004
```

### 问题2：API密钥无效

**症状**：
```
密钥验证失败: 401 Unauthorized
```

**解决**：
1. 检查原项目的`.env`文件中的API密钥
2. 确认密钥格式正确（智谱AI：`.mzTaQmnUZq4TH2fn`结尾）
3. 检查密钥是否过期

### 问题3：Python脚本执行失败

**症状**：
```
Error: 脚本执行失败 (code 1)
```

**解决**：
1. 确保Python已安装：`python --version`
2. 确保已安装依赖：`pip install python-docx`
3. 检查Python脚本路径是否正确

### 问题4：索引构建失败

**症状**：
```
索引构建失败: Cannot find module 'uuid'
```

**解决**：
```bash
cd E:\外包\教育系统智能体\后台管理系统\backend
npm install uuid
```

### 问题5：前端无法连接后端

**症状**：
```
Network Error
```

**解决**：
1. 确认后端服务正在运行（http://localhost:3004）
2. 检查前端代理配置（`vite.config.ts`）
3. 检查CORS设置

## 最佳实践

### 1. 文档准备

✅ **推荐**：
- 使用DOCX格式（结构清晰）
- 确保文档没有损坏
- 文档内容完整

❌ **避免**：
- 扫描版PDF（无法提取文本）
- 图片过多的文档
- 格式混乱的文档

### 2. 索引管理

✅ **推荐**：
- 定期备份索引
- 使用增量添加更新索引
- 删除不需要的文档

❌ **避免**：
- 频繁重建索引
- 同时运行多个构建任务
- 在高负载时重建索引

### 3. API使用

✅ **推荐**：
- 使用原项目的API密钥
- 合理设置请求频率
- 监控API使用量

❌ **避免**：
- 泄露API密钥
- 超出API限制
- 在代码中硬编码密钥
