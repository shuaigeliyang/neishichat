### 系统分工

**前台系统**（端口3000）：
- 用户：学生、老师
- 功能：问答、查询、表单下载

- 用户：系统管理员
- 功能：知识库管理、API配置、索引构建
- 输出：为前台系统构建索引

## 系统架构

### 前后台数据流向

```
管理员上传文档
    ↓
后台系统提取文档（Python脚本）
    ↓
智能分块（根据文档类型动态调整）
    ↓
批量向量化（调用智谱AI API）
    ↓
构建索引（保存到前台系统根目录）
    ↓
前台系统自动读取新索引
    ↓
学生/老师提问时使用最新索引
```

### 使用流程

#### 场景1：初次构建知识库

1. **访问后台管理系统**
   ```
   http://localhost:5173/knowledge-management
   ```

2. **配置API密钥**
   - 系统自动读取前台项目的API密钥
   - 或手动输入新的API密钥

3. **准备文档**
   ```
   将文档放到：E:\外包\教育系统智能体\相关文档\
   支持格式：DOCX, PDF, TXT, Markdown
   ```

4. **构建索引**
   - 在后台界面选择文档
   - 点击"构建新索引"
   - 查看实时进度

5. **验证索引**
   - 索引保存到：`E:\外包\教育系统智能体\retrieval_index.json`
   - 前台系统自动可以使用

#### 场景2：更新知识库

**增量添加**：
- 适合：新增少量文档
- 操作：上传新文档 → 点击"增量添加"

**替换索引**：
- 适合：文档内容有重大变更
- 操作：选择文档 → 点击"替换索引"
- 自动备份旧索引

#### 场景3：学生/老师使用问答

1. **访问前台系统**
   ```
   http://localhost:3000
   ```

2. **提问**
   ```
   学生：重修的条件是什么？
   系统：[返回基于文档的准确答案]
   ```
